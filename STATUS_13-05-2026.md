### Programação (8 tarefas executadas em sequência)

1. **`agente/sender.py`** — Refatorado pra adapter pattern. Suporta 4 backends:
   - `BaileysSender` (servidor Node local/AWS)
   - `ZAPISender` (Z-API brasileira)

   - `DryRunSender` (logga em `data/dryrun_log.jsonl`, nunca envia)

   - `GET /api/whatsapp/status` — status do sender configurado
   - `POST /api/aprovacao/preview` — gera candidatos sem enviar
   - `GET /api/aprovacao/pendentes` — vê o que tá esperando aprovação
   - `POST /api/aprovacao/confirmar` — aprova e envia subset
   - Workflow "manual approval" pra ganhar confiança antes de automatizar

3. **`agente/dashboard.py`** *(novo)* — Painel web em `/dashboard`:

   - Histórico das últimas 30 mensagens
   - Auto-refresh 30s. Vanilla JS, sem build.

4. **`agente/logging_config.py`** *(novo)* — Loguru estruturado:
   - Console colorido + arquivo rotativo (100MB, retenção 30 dias)
   - Fallback automático se `/var/log/cobranca/` não escrevível

5. **`agente/backup.py`** *(novo)* — Backup automático SQLite:
   - Diário às 23h (cron via APScheduler)
   - Mantém últimos 30, deleta antigos
   - Function `backup_state()` + CLI standalone

6. **`agente/main.py`** — Atualizado pra integrar logging + backup automático

7. **`agente/test_dryrun.py`** *(novo)* — Validação end-to-end:

   - Asserções: blacklist, idempotência, cobertura de etapas, números válidos
   - Output Markdown em `data/teste_dryrun_YYYYMMDD.md`
